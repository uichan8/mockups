# CLAUDE.md

## 프로젝트 개요

다양한 앱 아이디어를 **목업(mockup)**으로 빠르게 프로토타이핑하는 레포지토리. 각 목업은 독립된 디렉토리에 순수 HTML/CSS/JS 단일 파일로 구현하며, GitHub Pages로 바로 확인할 수 있다.

## 파일 구조

```
mockups/
├── CLAUDE.md                        # 개발 가이드 (이 파일)
├── index.html                       # GitHub Pages 진입점 (아코디언 목록)
├── template/                        # 새 목업용 보일러플레이트
│   ├── phone_template.html          #   폰 프레임 (375×812, 아이폰 스타일)
│   └── tablet_template.html         #   태블릿 프레임 (590×820, 아이패드 스타일)
├── gallering/                       # 사진 정리 앱 목업
│   └── photo_organizer.html
└── <새 목업>/                        # 목업마다 별도 디렉토리
    └── *.html
```

## 기술 스택

- 순수 HTML5 / CSS3 / Vanilla JavaScript
- 빌드 도구, 프레임워크 없음
- 각 목업은 **단일 HTML 파일**에 `<style>`과 `<script>` 포함 (인라인)
- 외부 CDN 라이브러리 사용 가능하나, 로컬 의존성은 금지

## 템플릿 사용 가이드

새 목업을 만들 때 `template/` 디렉토리의 보일러플레이트를 복사해서 시작한다.

### 제공 템플릿

| 템플릿 | 파일 | 해상도 | 용도 |
|--------|------|--------|------|
| 폰 프레임 | `template/phone_template.html` | 375×812 | 모바일 앱 목업 |
| 태블릿 프레임 | `template/tablet_template.html` | 590×820 | 태블릿 앱 목업 |

### 템플릿 공통 구성 요소

두 템플릿 모두 아래 요소가 포함되어 있다:

- **상태 바** — 시간, 배터리, 신호 아이콘
- **내비게이션 바** — 뒤로가기 + 타이틀 + 액션 버튼
- **테마 토글** — 라이트/다크 모드 전환
- **토스트 알림** — `showToast('메시지')` 함수로 호출
- **홈 인디케이터** — 하단 바

### 새 목업 만들기 (워크플로우)

```
1. 디렉토리 생성      mkdir <목업명>/
2. 템플릿 복사         cp template/phone_template.html <목업명>/<파일명>.html
3. 콘텐츠 구현         "앱 콘텐츠 영역" 주석 부분을 수정
4. 인덱스 등록         index.html의 projects 배열에 항목 추가
5. 브라우저 확인       파일을 직접 열어 동작 검증
```

### index.html 등록 방법

`index.html` 파일 내 `projects` 배열에 아래 형식으로 추가한다:

```javascript
{
  name: '프로젝트명',
  description: '한 줄 설명',
  pages: [
    {
      name: '페이지명',
      description: '페이지 설명',
      url: '<디렉토리>/<파일명>.html'
    }
  ]
}
```

## 개발 규칙

- **한국어** 사용 (커밋 메시지, 주석, 문서 모두 한국어)
- 커밋 메시지 접두사: `feat:`, `fix:`, `docs:`, `refactor:`, `style:`, `chore:`
- 별도 빌드/번들러 없이 브라우저에서 바로 실행 가능해야 함
- 새 목업 추가 시 별도 디렉토리를 만들고, `index.html` 목록에 링크 추가
- 템플릿 파일은 범용 보일러플레이트이므로, 특정 목업 전용 코드를 넣지 않는다

## PR 작성 규칙

- **제목**: `[목업명] 간결한 변경 요약` (예: `[Gallering] 조이스틱 감도 조절 기능 추가`)
  - 목업과 무관한 공통 변경은 `[공통]` 접두사 사용
- **본문 구성**:
  ```
  ## 변경 내용
  - 무엇을 왜 변경했는지 bullet으로 정리

  ## 스크린샷
  - UI 변경이 있으면 before/after 스크린샷 첨부

  ## 테스트
  - 브라우저에서 직접 확인한 항목 체크리스트
  ```
- **단위**: 하나의 PR은 하나의 목업 또는 하나의 주제에 집중
- **언어**: 제목·본문 모두 한국어
